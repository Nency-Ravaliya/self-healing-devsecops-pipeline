# AWS Services Used — Console Guide

This document lists every AWS service used in this project and how to find and verify it in the AWS Console.

---

## 1. AWS Lambda — AI Agent

**What it does**: Runs the AI Agent logic (Observe, Think, Act).

**How to find it**:
1. Go to [AWS Console](https://console.aws.amazon.com)
2. Search for **Lambda** in the top search bar
3. Click **Functions**
4. Find `ai-self-healing-agent`
5. Click on it to see:
   - **Configuration** tab: environment variables, memory, timeout
   - **Monitor** tab: invocation count, errors, duration
   - **Code** tab: view the deployed code

**Test it directly**:
- Click **Test** tab
- Create a test event with the sample payload from `docs/demo-walkthrough.md`
- Click **Test** to invoke it manually

---

## 2. Amazon Bedrock — AI Brain

**What it does**: Provides Claude 3 Haiku model for vulnerability analysis.

**How to find it**:
1. Search for **Bedrock** in the AWS Console
2. Click **Model catalog** in the left sidebar
3. Search for `Claude 3 Haiku` to find the model
4. Open it in the **Playground** to test it directly

> Note: The Model Access page has been retired. Foundation models (including Claude 3 Haiku) are now **automatically enabled** on first invocation. For Anthropic models, first-time users may be prompted to submit use case details before the model becomes available.

**Verify the Lambda can call Bedrock**:
```bash
aws logs tail /aws/lambda/ai-self-healing-agent --follow
# Look for: [THINK] AI Analysis complete.
```

**If Anthropic requires use case details**:
1. Go to Bedrock > Model catalog > Claude 3 Haiku
2. Click **Request access** and fill in business use case
3. Approval is usually instant

## 3. Amazon S3 — Storage

**What it does**: Stores generated SBOMs and AI analysis reports.

**How to find it**:
1. Search for **S3** in the AWS Console
2. Find bucket: `ai-devsecops-data-95aaa281`
3. Click on the bucket to see:
   - `sboms/` folder: CycloneDX SBOM files uploaded by CI
   - `reports/` folder: AI analysis reports generated by Lambda

**CLI alternative**:
```bash
aws s3 ls s3://ai-devsecops-data-95aaa281/
aws s3 ls s3://ai-devsecops-data-95aaa281/sboms/
aws s3 ls s3://ai-devsecops-data-95aaa281/reports/
```

## 4. AWS Secrets Manager — Secure Credentials

**What it does**: Stores the GitHub Personal Access Token securely.

**How to find it**:
1. Search for **Secrets Manager** in the AWS Console
2. Find secret: `github-token-ai-agent`
3. Click on it to see:
   - Secret metadata and rotation settings
   - Click **Retrieve secret value** to verify the token is stored

---

## 5. Amazon SNS — Notifications

**What it does**: Sends alert notifications when a fix is applied.

**How to find it**:
1. Search for **SNS** in the AWS Console
2. Click **Topics** in the left sidebar
3. Find `ai-devsecops-alerts`
4. Click on it to see:
   - Subscriptions (add your email here)
   - Published message history

**Subscribe your email**:
1. Click **Create subscription**
2. Protocol: **Email**
3. Endpoint: your email address
4. Confirm the subscription email you receive

---

## 6. Amazon EventBridge — Event Router

**What it does**: Routes AWS Inspector findings to the Lambda function automatically.

**How to find it**:
1. Search for **EventBridge** in the AWS Console
2. Click **Rules** in the left sidebar
3. Find `inspector-vulnerability-finding`
4. Click on it to see:
   - Event pattern (listens for `aws.inspector2` events)
   - Target: the `ai-self-healing-agent` Lambda

---

## 7. AWS IAM — Access Control

**What it does**: Controls permissions for Lambda, GitHub Actions, and OIDC auth.

**How to find it**:
1. Search for **IAM** in the AWS Console
2. Click **Roles** in the left sidebar
3. Find these roles:
   - `ai-self-healing-agent-role` — Lambda execution role
   - `ai-self-healing-agent-github-actions` — GitHub Actions OIDC role

4. Click each role to see the attached policies and permissions

**OIDC Provider**:
1. In IAM, click **Identity providers**
2. Find `token.actions.githubusercontent.com`
3. This enables keyless auth from GitHub Actions

---

## 8. Amazon CloudWatch — Logs and Monitoring

**What it does**: Stores all Lambda execution logs for debugging and audit.

**How to find it**:
1. Search for **CloudWatch** in the AWS Console
2. Click **Log groups** in the left sidebar
3. Find `/aws/lambda/ai-self-healing-agent`
4. Click on it to see all Lambda invocation logs

**View live logs via CLI**:
```bash
aws logs tail /aws/lambda/ai-self-healing-agent --follow
```
